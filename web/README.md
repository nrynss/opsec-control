# web/ — Astro + Svelte dashboard

The Common Operational Picture: Cerebro map, six live Cell panels, timeline, the
"Matrix" JSON feed, and the speed/throughput HUD (SPEC §15).

## Boundary (SPEC §16.1)

- Talks to **`internal/api` only** — nothing else.
- Types against the JSON Schemas in `internal/contracts/schemas/` (via the API).
- **No operational logic** lives here — the frontend visualizes state, it does
  not derive or own it.

## Scaffolding

The Astro project itself is the frontend Builder's lane and is intentionally not
generated by the repo scaffold. Initialize it here (Node pinned by the repo
`.nvmrc`):

```sh
npm create astro@latest .
npx astro add svelte
```

Then keep its `dist/`, `.astro/`, and `node_modules/` out of git (already in the
root `.gitignore`), and add Node `engines` to the generated `package.json` to
match `.nvmrc` (SPEC §19.1).

## Cross-platform (SPEC §19.2)

No shell one-liners in `package.json` scripts (`rm -rf`, `&&`, `VAR=x cmd` break
on Windows) — use `rimraf`/`cross-env` or Node scripts.
